# T3-13: Make HTTP client timeout configurable

## Problem

The shared HTTP client in `core/llm/providers/http_client.py` had hardcoded 600-second (10 minute) timeouts:

```python
timeout=httpx.Timeout(connect=10.0, read=600.0, write=600.0, pool=600.0),
```

This couldn't be configured per-environment (e.g., shorter timeouts in development, longer for slow models).

## Solution

Added environment variable support via `LLM_HTTP_TIMEOUT`:

```python
import os

LLM_HTTP_TIMEOUT = float(os.environ.get('LLM_HTTP_TIMEOUT', '600.0'))

shared_http_client = httpx.Client(
    ...
    timeout=httpx.Timeout(connect=10.0, read=LLM_HTTP_TIMEOUT, write=LLM_HTTP_TIMEOUT, pool=LLM_HTTP_TIMEOUT),
)
```

## Changes

- **File**: `core/llm/providers/http_client.py`
  - Added `import os`
  - Added `LLM_HTTP_TIMEOUT` constant reading from environment variable
  - Updated `httpx.Timeout` to use the configurable value for read, write, and pool timeouts
  - Connect timeout remains fixed at 10s (connection establishment should be fast)

## Backward Compatibility

Default remains 600 seconds (10 minutes), preserving existing behavior.

## Usage

Set environment variable to configure:
```bash
export LLM_HTTP_TIMEOUT=300  # 5 minutes
```

## Test

No unit test required per spec - this is an optional environment variable configuration. The change is mechanical and backward compatible.
