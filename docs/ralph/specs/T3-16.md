# T3-16: Consolidate duplicated DB path logic

## Problem

The same DB path detection logic is duplicated in 6+ locations:

1. `flask_app/config.py:108` — `get_db_path()` (canonical source)
2. `flask_app/routes/game_routes.py:456` — `_get_db_path()`
3. `flask_app/routes/admin_dashboard_routes.py:20` — `_get_db_path()`
4. `core/llm/tracking.py:120` — `_get_default_db_path()`
5. `flask_app/routes/image_routes.py:49,402` — inline versions
6. `bin/seed_personalities.py:26` — `get_db_path()`
7. `populate_personalities_db.py:25-28` — inline version

All have the same pattern: check if `/app/data` exists (Docker), else use project root.

## Changes Made

### 1. `flask_app/routes/game_routes.py`
- Deleted local `_get_db_path()` function
- Added import: `from flask_app.config import get_db_path`
- Changed usage from `_get_db_path()` to `get_db_path()`

### 2. `flask_app/routes/admin_dashboard_routes.py`
- Deleted local `_get_db_path()` function
- Added import: `from flask_app.config import get_db_path`
- Changed usage from `_get_db_path()` to `get_db_path()`

### 3. `flask_app/routes/image_routes.py`
- Modified `_get_persistence()` at line 49 to use `from flask_app.config import get_db_path`
- Modified `get_reference_image_data_url()` at line 402 to use imported `get_db_path()`

### 4. `core/llm/tracking.py`
- Kept local `_get_default_db_path()` method with a comment pointing to the canonical version
- Reason: `core/llm` is a low-level module that should not depend on `flask_app` to avoid circular imports

### 5. Standalone scripts (`bin/seed_personalities.py`, `populate_personalities_db.py`)
- Kept local versions with comments pointing to canonical source
- Reason: These are standalone scripts that run independently and shouldn't require full flask_app initialization

## Test

No new test file needed — this is a pure refactor that consolidates duplicated code. The existing test suite validates that the application still works correctly after consolidation.
