# T1-19: Fix SECRET_KEY — require in prod, stable default in dev

## Problem

`flask_app/config.py:18` used `os.urandom(32).hex()` as the default SECRET_KEY:

```python
SECRET_KEY = os.environ.get('SECRET_KEY', os.urandom(32).hex())
```

This generates a new random key on every process restart when no `SECRET_KEY` environment variable is set. In production, this invalidates all existing sessions (cookies, CSRF tokens) on every deploy or restart.

## What Changed

Replaced the single-line SECRET_KEY assignment with environment-aware logic:

- **Development mode** (`FLASK_ENV=development` or `FLASK_DEBUG=1`): Uses a stable default string `'dev-secret-key-not-for-production'`, or the `SECRET_KEY` env var if explicitly set. This prevents session loss during local development restarts.
- **Production mode**: Requires `SECRET_KEY` to be set as an environment variable. Raises `RuntimeError` with a helpful generation command if missing.

## What the Tests Verify

`tests/test_config_secret_key.py` — 7 tests:

1. Dev mode uses stable default when no env var set
2. Dev default is stable across module reloads (not random)
3. Dev mode respects explicit SECRET_KEY env var
4. FLASK_DEBUG=1 is treated as development mode
5. Production mode raises RuntimeError without SECRET_KEY
6. Error message includes `secrets.token_hex` generation command
7. Production mode uses SECRET_KEY from environment when set
