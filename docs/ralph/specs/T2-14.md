# T2-14: Fix `get_celebrities()` shuffle mutation

## Problem
`get_celebrities()` in `poker/utils.py` assigns `celebrities_list = CELEBRITIES_LIST`, which creates a reference (not a copy). Calling `random.shuffle(celebrities_list)` mutates the module-level constant in place, permanently altering the order for all subsequent callers.

## Fix
Replaced the function body:
- Shuffled case: use `random.sample(CELEBRITIES_LIST, len(CELEBRITIES_LIST))` which returns a new list without mutating the original.
- Non-shuffled case: return `list(CELEBRITIES_LIST)` â€” a defensive copy.

## Test
`tests/test_triage_t2_14.py` verifies:
1. `CELEBRITIES_LIST` is not mutated after calling `get_celebrities(shuffled=True)`
2. Shuffled call returns all celebrities (same set, different order possible)
3. Non-shuffled call returns original order
4. Returned list is a different object (not a reference to the constant)
