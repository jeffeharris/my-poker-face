# T2-12: Remove debug console.log statements from production

## Problem

120+ raw `console.*` statements across 38+ React source files:
- ~48 `console.log` (debug noise firing on every WebSocket event)
- ~62 `console.error` (legitimate error handling)
- ~8 `console.warn`
- ~2 `console.debug`

Debug logs cluttered the browser console and potentially exposed game state data in production. An existing `config.ENABLE_DEBUG` flag existed but was unused for logging.

## What Changed

### Step 1: Logger utility (already existed)
`react/react/src/utils/logger.ts` was already created (likely by a prior task). It provides:
- `logger.log()` / `logger.debug()` — gated behind `config.ENABLE_DEBUG`
- `logger.warn()` / `logger.error()` — always output (production-relevant)

### Step 2: Deleted pure debug noise (~25 statements)
Removed outright from source files (not converted):
- WebSocket routine logs in `usePokerGame.ts` (19 statements: connect, disconnect, game state updates, player turns, winner announcements, etc.)
- `[ExperimentDesigner]` flow tracing (5 statements)
- `[ExperimentChat]` debug logs (5 statements)
- `[PWA]` event logs (2 statements)
- `GameSelector` fetch logs (2 statements)
- `PokerTable.tsx` human turn debug object (1 block)
- `GameContext.tsx` WebSocket routine logs (4 statements)

### Step 3: Converted remaining console.log to logger.log (~8 statements)
- Debug panel WebSocket updates (`DebugPanel.tsx`, `ElasticityDebugPanel.tsx`)
- `CSSDebugger.tsx` debug output (converted to `logger.log`/`logger.warn`)

### Step 4: Converted console.error/warn/debug to logger equivalents (~70 statements)
All `console.error`, `console.warn`, and `console.debug` calls across 39 files converted to `logger.error`, `logger.warn`, and `logger.debug` respectively.

### Step 5: Verification
Final grep confirms zero raw `console.*` calls outside of:
- `logger.ts` itself (expected — it wraps `console.*`)
- `ErrorBoundary.test.tsx` (expected — mocks `console.error` for test suppression)
